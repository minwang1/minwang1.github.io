git command cheat sheet#

  • HEAD指向当前操作的地方
  • 相对引用：~Num 是num笔提交的引用，^指向1比提交引用。^ == ~1
  • HEAD~1 指向HEAD之前一笔提交，HEAD~2指向HEAD之前2笔提交
  • HEAD^2 比较特殊，指向HEAD最近的父合并节点的第二个合并提交。
  • git describe <ref>

命令用来描述离你最近的锚点 (tag), 它输出的结果是这样的：<tag>_<numCommits>_g<hash>

  • git tag <tagName> <ref>

在ref位置添加一个锚点

  • git rebase <targetBranch> 将当前分支合并到目标分支上，并且切换到合并后的分支上。

  • git rebase -i 交互式rebase，可以合并多比提交

  • git reset <ref> 本地仓库代码回退到<ref>指定的提交地方

  • git revert <ref> 删除到<ref>提交，会产生一个删除代码的提交

==================远程操作=========================

  • <remote> 远程仓库名称，默认origin

  • git push <remote> <place>

git push origin master

把这个命令翻译过来就是：

切到本地仓库中的“master”分支，获取所有的提交，再到远程仓库“origin”中找到“master”分支，将远程仓库中没有的提交记录都添加上去，搞定之后告诉我。

  • git push origin <source>:<destination>

指定source分支，推送到origin仓库的destination分支上。

  • git push origin :side

删除远端side分支

  • git fetch <remote> <place> 将remote仓库上的place分支检出到本地的place分支上

  • git fetch <remote> <source>:<place> 将remote仓库上的source分支检出到本地的place分支上

  • git pull origin master

相当于： git fetch origin master; git merge o/master 通过指定 master 我们更新了 o/master。然后将 o/master merge 到我们的检出位置， 无论我们当前检出的位置是哪

  • git branch -f <branchName> <ref> 例如 git branch -f master HEAD~2

强迫master分支指向HEAD之前的2笔提交的位置。可用于切换分支指向位置。

  • git pull –rebase

相当于执行一次git fetch,再执行一次git rebase
